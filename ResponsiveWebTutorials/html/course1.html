BOOTSRAPPING

A process by which simple language is used to translate more complicated program, which in turn may handle an even more complicated program.
Bootstrapping is used to produce a self hosting compiler, a self hosting compiler is a compiler which compile its own source code.
Bootstrap compiler is used to compile the compiler & then you can use this compiler to compile everything else as well as future version of itself.

A technique to produce a chicken without any chicken egg, using any existing egg.
In a more technical fashion, it’s a method to produce self hosting compiler, that is, a compiler whose source code is written in the language it compiles. E.g. a Pascal compiler written in Pascal, C compiler written in C, etc.
The procedure is to first write the target compiler in any language with existing implementation, then convert it (or write it from scratch) to the language that the compiler compiles.
The first Pascal compiler was written in Fortran, the first C compiler (I guess) was written in B(CPL?), the first Java compiler was written in C, and so on. What’s important, the first compiler source code was never used anymore once the resulting compiler can compile its own source code. It’s also possible to bootstrap for the same language, but different compiler. Free Pascal was written in Turbo Pascal and it took two years before it can compile its own source code, leaving Turbo Pascal forever. In this case, the first compiler source code is left intact (and become the base for further development) since it’s written in the same language.

In computer science, bootstrapping is the technique for producing a self-compiling compiler — that is, compiler (or assembler) written in the source programming language that it intends to compile. An initial core version of the compiler (the bootstrap compiler) is generated in a different language (which could be assembly language); successive expanded versions of the compiler are developed using this minimal subset of the language.

A typical bootstrap process work in three or four stages:
Stage 0: preparing an environment for the bootstrap compiler to work with.
Stage 1: the bootstrap compiler is produced.
Stage 2: a full compiler is produced by the bootstrap compiler.
Stage 3: a full compiler is produced by the stage 2 full compiler.
The full compiler is built twice in order to compare the outputs of the two stages. If they are different, either the bootstrap or the full compiler contains a bug
